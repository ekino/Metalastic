# Metalastic Logo Design Guidelines

## Brand Identity

**Library Purpose**: Metalastic is a Kotlin-first library that generates type-safe metamodels for Elasticsearch documents, enabling fluent, compile-time validated queries.

**Core Values**:
- Type Safety & Precision
- Elasticsearch Integration
- Kotlin-first Development
- Code Generation & Metamodels
- Professional & Reliable

## Name Analysis

**"Metalastic"** = **Meta** (metamodel) + **Elastic** (Elasticsearch)
- Pronunciation: "MET-uh-las-tik"
- Emphasis on "META" for metamodel concept
- "ELASTIC" connects to Elasticsearch ecosystem

## Visual Concepts

### Primary Concept: "Structured Elasticity"
- **Core Idea**: Flexible yet structured framework
- **Visual**: Geometric grid or framework that can stretch/adapt while maintaining structural integrity
- **Symbolism**: Type safety (structure) + Elasticsearch flexibility (elasticity)

### Secondary Concept: "Metamodel Transformation"
- **Core Idea**: Source code transforming into structured metamodels
- **Visual**: Flow/transformation from organic shapes to geometric precision
- **Symbolism**: Annotation processing converting classes to type-safe metamodels

### Tertiary Concept: "Interconnected Architecture"
- **Core Idea**: Connected nodes representing field relationships
- **Visual**: Network/molecular structure with clean geometric nodes
- **Symbolism**: Document fields mapped to metamodel properties

## Typography Guidelines

### Primary Typeface Style
- **Category**: Modern Geometric Sans-serif
- **Characteristics**: Clean, technical, readable
- **Weight**: Medium to Bold for main logo
- **Examples**: Geometric fonts like Montserrat, Proxima Nova, or similar

### Logo Text Treatment
- **Full Name**: "Metalastic"
- **Emphasis**: Slightly larger or bold "META" with normal weight "lastic"
- **Alternative**: "Meta" + "lastic" with visual separator
- **Tagline**: "Type-safe Elasticsearch metamodels" (smaller, lighter weight)

## Color Palette

### Primary Colors
- **Deep Steel Blue**: `#2563EB` (primary brand color - trust, technical precision)
- **Elasticsearch Orange**: `#F59E0B` (accent - industry connection)
- **Metamodel Gray**: `#6B7280` (secondary - professional, structured)

### Supporting Colors
- **Pure White**: `#FFFFFF` (backgrounds, contrast)
- **Dark Charcoal**: `#1F2937` (text, dark themes)
- **Kotlin Purple**: `#7C3AED` (optional accent - language connection)

### Color Combinations
- **Primary**: Steel Blue + White
- **Accent**: Steel Blue + Orange highlights
- **Monochrome**: Dark Charcoal + White
- **Dark Mode**: White/Light Gray on Dark Charcoal

## Icon Design Specifications

### Core Symbol Ideas

#### Option A: Elastic Grid Framework
- Geometric grid pattern forming an "M" shape
- Grid lines that curve slightly (showing elasticity)
- Intersections highlighted with small nodes/dots
- Clean, technical aesthetic

#### Option B: Metamodel Molecule
- Atomic/molecular structure with 5-7 connected nodes
- Central node larger (representing source document)
- Surrounding nodes (representing metamodel fields)
- Clean connecting lines, geometric precision

#### Option C: Transformation Arrow
- Stylized "M" with transformation elements
- Left side: organic/flowing (source code)
- Right side: geometric/structured (metamodel)
- Arrow or flow indicating generation process

### Technical Specifications
- **Style**: Geometric, clean, professional
- **Line Weight**: Medium thickness (not too thin for small sizes)
- **Corners**: Slightly rounded for modern feel
- **Proportions**: Square aspect ratio preferred for icon versions
- **Scalability**: Must work from 16px to 512px

## Logo Variations Required

### 1. Full Logo (Horizontal)
- Icon + "Metalastic" text side by side
- For headers, documentation, marketing materials
- Aspect ratio: approximately 3:1 to 4:1

### 2. Stacked Logo (Vertical)
- Icon above "Metalastic" text
- For square/vertical spaces
- Aspect ratio: approximately 1:1 to 1:1.5

### 3. Icon Only
- Just the symbol/icon without text
- For favicons, app icons, social media profiles
- Perfect square: 1:1 aspect ratio

### 4. Text Only (Wordmark)
- "Metalastic" text with distinctive typography
- For minimal applications, code documentation
- Horizontal format

### 5. Monogram
- Stylized "M" letterform
- Ultra-minimal applications
- Square format

## Usage Guidelines

### Minimum Sizes
- **Full Logo**: 120px width minimum
- **Icon Only**: 16px minimum (favicon)
- **Text Only**: 80px width minimum

### Clear Space
- Minimum clear space around logo: 0.5x the height of the icon
- No other elements within this space
- Ensures visual breathing room

### Background Applications
- **Light Backgrounds**: Use primary colors (steel blue, orange)
- **Dark Backgrounds**: Use white or light gray versions
- **Colored Backgrounds**: Use white or dark versions for contrast
- **Photographic Backgrounds**: Add subtle backdrop for legibility

## Style Characteristics

### Visual Personality
- **Professional**: Suitable for enterprise documentation
- **Technical**: Appeals to developers and engineers
- **Modern**: Contemporary design language
- **Trustworthy**: Conveys reliability and precision
- **Approachable**: Not overly complex or intimidating

### Design Principles
- **Simplicity**: Clean, uncluttered design
- **Scalability**: Works at any size
- **Memorability**: Distinctive and recognizable
- **Relevance**: Clearly connected to metamodels and Elasticsearch
- **Timelessness**: Won't look dated in 5+ years
